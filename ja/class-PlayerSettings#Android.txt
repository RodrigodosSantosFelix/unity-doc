<div id='wikitext'>
<h1>Player Settings</h1>
<p><span class='doc-keyword'>Player Settings</span> is where you define various parameters (platform specific) for the final game that you will build in Unity. Some of these values for example are used in the <span class='doc-keyword'>Resolution Dialog</span> that launches when you open a standalone game, others are used by XCode when building your game for the iOS devices, so it's important to fill them out correctly.
</p>
<p class='vspace'>To see the Player Settings choose <span class='doc-menu'>Edit-&gt;Project Settings-&gt;Player</span> from the menu bar.
</p>
<div class='vspace'></div><div><img src='http://docwiki.unity3d.co.jp/uploads/Main/PlayerSetInspTop.png' alt='' title='' /><br /><em>Global Settings that apply to any project you create.</em></div>
<div class='vspace'></div><table width='100%'><tr ><th  align='left'>Property:</th><th  align='left'>Function:</th></tr>
<tr ><th  align='left' colspan='2'>Cross-Platform Properties</th></tr>
<tr ><td  align='left'><span class='doc-prop'>Company Name</span></td><td  align='left'>The name of your company. This is used to locate the preferences file.</td></tr>
<tr ><td  align='left'><span class='doc-prop'>Product Name</span></td><td  align='left'>The name that will appear on the menu bar when your game is running and is used to locate the preferences file also.</td></tr>
<tr ><td  align='left'><span class='doc-prop'>Default Icon</span></td><td  align='left'>Default icon the application will have on every platform (You can override this later for platform specific needs).</td></tr>
</table>

<h2>Web-Player</h2>
<h4><em>Resolution And Presentation</em></h4>
<div><img src='http://docwiki.unity3d.co.jp/uploads/Main/PlayerSetWebResPres.png' alt='' title='' /></div>
<div class='vspace'></div><table width='100%'><tr ><th  align='left'>Property:</th><th  align='left'>Function:</th></tr>
<tr ><td  align='left' colspan='2'><b><em>Resolution</em></b></td></tr>
<tr ><td  align='left'><span class='doc-prop'>Default Screen Width</span></td><td >Screen Width the player will be generated with.</td></tr>
<tr ><td  align='left'><span class='doc-prop'>Default Screen Height</span></td><td >Screen Height the plater will be generated with.</td></tr>
<tr ><td  align='left'><span class='doc-prop'>Run in background</span></td><td >Check this if you dont want to stop executing your game if the player looses focus.</td></tr>
<tr ><td  align='left'><b><em>WebPlayer Template</em></b></td><td >For more information you should check the <a class='wikilink' href='http://docwiki.unity3d.co.jp/index.php?n=Main.UsingWebPlayertemplates'>"Using WebPlayer templates page"</a>, note that for each built-in and custom template there will be an icon in this section.</td></tr>
</table>
<div class='vspace'></div><h4><em>Icon</em></h4>
<div><img src='http://docwiki.unity3d.co.jp/uploads/Main/PlayerSetWebIcon.png' alt='' title='' /></div>
<p class='vspace'>Icons don't have any meaning for most webplayer builds but they are needed for Native Client builds used as Chrome applications. You can set these icons here.
</p>
<div class='vspace'></div><h4><em>Other Settings</em></h4>
<div><img src='http://docwiki.unity3d.co.jp/uploads/Main/PlayerSetWebOther355.png' alt='' title='' /></div>
<div class='vspace'></div><table width='100%'><tr ><th  align='left'>Property:</th><th  align='left'>Function:</th></tr>
<tr ><td  align='left'><b><em>Rendering</em></b></td></tr>
<tr ><td  align='left'><span class='doc-prop'>Rendering Path</span></td><td >This property is shared between Standalone and WebPlayer content.</td></tr>
<tr ><td  align='left'><span  style='padding-left: 1.5em;'><span class='doc-prop'>Vertex Lit</span></span></td><td >Lowest lighting fidelity, no shadows support. Best used on old machines or limited mobile platforms.</td></tr>
<tr ><td  align='left'><span  style='padding-left: 1.5em;'><span class='doc-prop'>Forward with Shaders</span></span></td><td >Good support for lighting features; limited support for shadows.</td></tr>
<tr ><td  align='left'><span  style='padding-left: 1.5em;'><span class='doc-prop'>Deferred Lighting</span></span></td><td >Best support for lighting and shadowing features, but requires certain level of hardware support. Best used if you have many realtime lights. Unity Pro only.</td></tr>
<tr ><td ><span class='doc-prop'>Color Space</span></td><td >The color space to be used for rendering</td></tr>
<tr ><td ><span  style='padding-left: 1.5em;'><span class='doc-prop'>GammaSpace Rendering</span></span></td><td >Rendering is gamma-corrected</td></tr>
<tr ><td ><span  style='padding-left: 1.5em;'><span class='doc-prop'>Linear Rendering Hardware Sampling</span></span></td><td >Rendering is done in linear space</td></tr>
<tr ><td  align='left'><span class='doc-prop'>Static Batching</span></td><td >Set this to use Static batching on your build (Inactive by default in webplayers). Unity Pro only.</td></tr>
<tr ><td  align='left'><span class='doc-prop'>Dynamic Batching</span></td><td  align='left'>Set this to use Dynamic Batching on your build (Activated by default).</td></tr>
<tr ><td  align='left' colspan='2'><b><em>Streaming</em></b></td></tr>
<tr ><td  align='left'><span class='doc-prop'>First Streamed Level</span></td><td >If you are publishing a Streamed Web Player, this is the index of the first level that will have access to all Resources.Load assets.</td></tr>
<tr ><td  colspan='2'><b><span class='doc-prop'>Optimization</span></b></td></tr>
<tr ><td ><span class='doc-prop'>Optimize Mesh Data</span></td><td >Remove any data from meshes that is not required by the material applied to them (tangents, normals, colors, UV).</td></tr>
<tr ><td ><span class='doc-prop'>Debug Unload Mode</span></td><td >Output debugging information regarding Resources.UnloadUnusedAssets.</td></tr>
<tr ><td ><span  style='padding-left: 1.5em;'><span class='doc-prop'>Disabled</span></span></td><td >Don't output any debug data for UnloadUnusedAssets.</td></tr>
<tr ><td ><span  style='padding-left: 1.5em;'><span class='doc-prop'>Overview only</span></span></td><td >Minimal stats about UnloadUnusedAssets usage.</td></tr>
<tr ><td ><span  style='padding-left: 1.5em;'><span class='doc-prop'>Full (slow)</span></span></td><td >Output overview stats along with stats for all affected objects. This option can slow down execution due to the amount of data being displayed.</td></tr>
</table>

<h4><em>Resolution And Presentation</em></h4>

<tr ><td  align='left'><span class='doc-prop'>Capture Single Screen</span></td><td >If enabled, standalone games in fullscreen mode will not darken the secondary monitor in multi-monitor setups.</td></tr>
<tr ><td  align='left' colspan='2'><span class='doc-prop'>DisplayResolution Dialog</span></td></tr>
<tr ><td  align='left'><span  style='padding-left: 1.5em;'><span class='doc-prop'>Disabled</span></span></td><td >No resolution dialog will appear when starting the game.</td></tr>

<tr ><td  align='left'><span class='doc-prop'>Use Player Log</span></td><td  align='left'>Write a log file with debugging information. If you plan to submit your application to the Mac App Store you will want to leave this option un-ticked.  Ticked is the default.</td></tr>
<tr ><td  align='left'><span class='doc-prop'>Mac App Store Validation</span></td><td  align='left'>Enable receipt validation for the Mac App Store.</td></tr>
<tr ><td  align='left'><span class='doc-prop'>Supported Aspect Ratios</span></td><td >Aspect Ratios selectable in the Resolution Dialog will be monitor-supported resolutions of enabled items from this list.</td></tr>
</table>

<div class='vspace'></div><h1>Details</h1>
<a class='desktopRef' id='desktopRef' href="javascript:ShowHideDiv('desktopRef','specific-desktop','Desktop');"><img src="./uploadks/Main/PlatformArrowDown.png"><h1>Desktop</h1> </a>
<div class='specific-desktop' style='display: block;' id='specific-desktop' > 
<p>The Player Settings window is where many technical preference defaults are set. See also <a class='wikilink' href='http://docwiki.unity3d.co.jp/index.php?n=Main.class-QualitySettings'>Quality Settings</a> where the different graphics quality levels can be set up.
</p>
<div class='vspace'></div><h3>Publishing a web player</h3>
<p><span class='doc-prop'>Default Web Screen Width</span> and <span class='doc-prop'>Default Web Screen Height</span> determine the size used in the html file. You can modify the size in the html file later.
</p>
<p class='vspace'><span class='doc-prop'>Default Screen Width</span> and <span class='doc-prop'>Default Screen Height</span> are used by the Web Player when entering fullscreen mode through the context menu in the Web Player at runtime.
</p>
<div class='vspace'></div><h3>Customizing your Resolution Dialog</h3>
<div><img src='http://docwiki.unity3d.co.jp/uploads/Main/Resolution-GameLauncher.png' alt='' title='' /><br /><em>The Resolution Dialog, presented to end-users</em></div>
<p class='vspace'>You have the option of adding a custom banner image to the Screen Resolution Dialog in the Standalone Player. The maximum image size is 432 x 163 pixels. The image will not be scaled up to fit the screen selector. Instead it will be centered and cropped.
</p>
<div class='vspace'></div><h3>Publishing to Mac App Store</h3>
<p><span class='doc-prop'>Use Player Log</span> enables writing a log file with debugging information. This is useful to find out what happened if there are problems with your game. When publishing games for Apple's Mac App Store, it is recommended to turn this off, because Apple may reject your submission otherwise. See <a class='wikilink' href='http://docwiki.unity3d.co.jp/index.php?n=Main.LogFiles'>this manual page</a> for further information about log files.
</p>
<p class='vspace'><span class='doc-prop'>Use Mac App Store Validation</span> enables receipt validation for the Mac App Store. If this is enabled, your game will only run when it contains a valid receipt from the Mac App Store. Use this when submitting games to Apple for publishing on the App Store. This prevents people from running the game on any computer then the one it was purchased on. Note that this feature does not implement any strong copy protection. In particular, any potential crack against one Unity game would work against any other Unity content. For this reason, it is recommended that you implement your own receipt validation code on top of this using Unity's plugin feature. However, since Apple requires plugin validation to initially happen before showing the screen setup dialog, you should still enable this check, or Apple might reject your submission. 
</p></div>
<div class='vspace'></div><a class='iosRef' id='iosRef' href="javascript:ShowHideDiv('iosRef','specific-ios', 'iOS');"><img src="./uploads/Main/PlatformArrowDown.png"><h1>iOS</h1></a>
<div class='specific-ios' style='display: block;' id='specific-ios' > 
<h3>Bundle Identifier</h3>
<p>The <span class='doc-prop'>Bundle Identifier</span> string must match the provisioning profile of the game you are building.  The basic structure of the identifier is <span class='doc-prop'>com.CompanyName.GameName</span>. This structure may vary internationally based on where you live, so always default to the string provided to you by Apple for your Developer Account.  Your GameName is set up in your provisioning certificates, that are manageable from the Apple iPhone Developer Center website.  Please refer to the <a class='urllink' href='http://developer.apple.com/iphone/' rel='nofollow'>Apple iPhone Developer Center website</a> for more information on how this is performed.
</p>
<div class='vspace'></div><h3>Stripping Level (Pro-only)</h3>
<p>Most games don't use all necessary dlls.  With this option, you can strip out unused parts to reduce the size of the built player on iOS devices.  If your game is using classes that would normally be stripped out by the option you currently have selected, you'll be presented with a Debug message when you make a build.
</p>
<div class='vspace'></div><h3>Script Call Optimization</h3>
<p>A good development practice on iOS is to never rely on exception handling (either internally or through the use of try/catch blocks).  When using the default <span class='doc-prop'>Slow and Safe</span> option, any exceptions that occur on the device will be caught and a stack trace will be provided.  When using the <span class='doc-prop'>Fast but no Exceptions</span> option, any exceptions that occur will crash the game, and no stack trace will be provided.  However, the game will run faster since the processor is not diverting power to handle exceptions.  When releasing your game to the world, it's best to publish with the <span class='doc-prop'>Fast but no Exceptions</span> option.
</p></div>
<div class='vspace'></div><a class='androidRef' id='androidRef' href="javascript:ShowHideDiv('androidRef','specific-android', 'Android');"><img src="./uploads/Main/PlatformArrowDown.png"><h1>Android</h1></a>
<div class='specific-android' style='display: block;' id='specific-android' > 
<h3>Bundle Identifier</h3>
<p>The <span class='doc-prop'>Bundle Identifier</span> string is the unique name of your application when published to the Android Market and installed on the device.  The basic structure of the identifier is <span class='doc-prop'>com.CompanyName.GameName</span>, and can be chosen arbitrarily.  In Unity this field is shared with the iOS Player Settings for convenience.
</p>
<div class='vspace'></div><h3>Stripping Level (Pro-only)</h3>
<p>Most games don't use all the functionality of the provided dlls.  With this option, you can strip out unused parts to reduce the size of the built player on Android devices.
</p></div>
